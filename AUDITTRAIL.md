## Session 1 ‚ÄĒ Planning & Initialisation

### Decision log

| # | Decision | Rationale |
|---|----------|-----------|
| 1 | Next.js 16.2.6 (App Router, TypeScript strict) | `create-next-app` resolved to latest ‚ÄĒ spec said 14 but 16 is backward-compatible for our use |
| 2 | Tailwind v4 (not v3) | Installed automatically; brand tokens moved to `globals.css @theme` block instead of `tailwind.config.ts` |
| 3 | `middleware.ts` renamed to `proxy.ts` | Next.js 16 deprecated `middleware` file convention in favour of `proxy` |
| 4 | Leaflet (not Google Maps) for contractor map | No API key required; free tiles; swap later if needed |
| 5 | URL search params for gallery filters | Shareable links; no Zustand needed |
| 6 | `users.role` column (`customer` / `admin`) | Simpler than a separate `admin_users` table |
| 7 | `MockPaymentProvider` interface | Lets full checkout ‚Üí download flow work without PayMongo/Stripe keys |
| 8 | GSAP only for hero parallax + stats counter | Heavy library ‚ÄĒ isolated to where it earns its weight |
| 9 | Framer Motion for all component-level animation | Lighter, React-idiomatic |
| 10 | Sonner for toasts | Preferred over react-hot-toast; shadcn-compatible |
| 11 | npm (not pnpm/yarn) | Default from create-next-app |
| 12 | `supabase/migrations/` tracked in git | SQL-as-code; Supabase MCP used to push (no CLI personal token needed) |
